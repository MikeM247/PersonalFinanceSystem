# Definition of Done

A story is considered complete when:

## Functional
- Acceptance criteria are fully satisfied
- No known functional defects

## Code Quality
- Code follows project standards
- No unnecessary complexity introduced

## Testing
- Unit tests exist for business logic
- Integration tests added where applicable
- All tests pass

## Build & CI
- Build succeeds
- Lint and type checks pass

## Documentation
- Any required documentation updated
- API changes documented if applicable

## Observability (if applicable)
- Errors are handled appropriately
- Logging added for critical paths

## Scope
- Only the intended story is implemented
